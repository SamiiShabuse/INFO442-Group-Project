# Data Folder

This folder stores the datasets used by the project. It is organized by pipeline stage so teammates can tell which files are raw source downloads, cleaned source data, integrated data, feature data, and model results.

## Folder Structure

```text
data/
  raw/
    fred/
    wikipedia/
    yfinance/
  processed/
    source_data/
      fred/
      wikipedia/
      yfinance/
    integrated/
    features/
    modeling/
```

## Raw Data

`raw/` contains original source-level files before project cleaning and integration.

- `raw/fred/` contains FRED macroeconomic and risk series.
- `raw/wikipedia/` contains the S&P 500 constituent metadata pulled from Wikipedia.
- `raw/yfinance/` contains market price, return, and summary files downloaded from Yahoo Finance through `yfinance`.

## Processed Source Data

`processed/source_data/` contains cleaned files for each source before they are merged together.

- `processed/source_data/fred/`
- `processed/source_data/wikipedia/`
- `processed/source_data/yfinance/`

These folders are generated by the notebooks in `notebooks/01_source_data/`.

## Project-Level Processed Data

- `processed/integrated/` contains the combined market, metadata, and macro datasets.
- `processed/features/` contains the final feature-engineered modeling dataset.
- `processed/modeling/` contains saved model predictions and metrics.

Each model output folder under `processed/modeling/` should contain:

- `test_predictions.csv`
- `metrics.csv`

## Notes

Do not put notebooks, source code, reports, dashboard files, or slides in this folder. Those belong in `notebooks/`, `src/`, `reports/`, or `dashboard/`.
